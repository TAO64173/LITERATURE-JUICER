## 2. Conflict Analysis

### CRITICAL: Color Palette Mismatch

| Element           | main.css (current)       | styles.css (new)         | Conflict |
|-------------------|--------------------------|--------------------------|----------|
| Primary color     | #2563EB (blue)           | #0D9488 (teal)           | HIGH     |
| Accent color      | #7C3AED (purple)         | #F59E0B (amber)          | HIGH     |
| Gradient text     | blue→purple              | not defined              | MEDIUM   |
| Progress bar      | blue→purple gradient     | not defined              | MEDIUM   |
| Upload zone hover | blue tints               | not defined              | MEDIUM   |
| Status badges     | blue/green/amber/red     | not defined              | LOW      |

### HIGH: .card Class Collision

Both files define `.card`:
- main.css: `background: #fff; border-radius: 16px; border: 1px solid rgba(0,0,0,0.05); box-shadow: 0 1px 2px rgba(0,0,0,0.04);`
- styles.css: `background: var(--bg-surface); border-radius: var(--radius-xl); border: 1px solid var(--border-subtle); box-shadow: var(--shadow-sm); padding: var(--space-lg);`

The styles.css version adds `padding: var(--space-lg)` which will break existing card layouts that rely on Tailwind padding classes (e.g., `card p-8`, `card p-7`).

### MEDIUM: .card-hover vs .card hover behavior

- main.css `.card-hover:hover`: `transform: translateY(-4px); box-shadow: 0 8px 30px...`
- styles.css `.card:hover`: `transform: translateY(-2px); box-shadow: var(--shadow-lg);`

Different hover lift distances and shadow intensities.

### MEDIUM: Missing Component Tokens

main.css components have no token equivalents in styles.css:
- .btn-primary, .btn-ghost (buttons)
- .upload-zone, .drag-over (upload area)
- .pdf-thumb, .pdf-thumb-* (file grid)
- .result-table, .result-table-* (data table)
- .toast, .toast-* (notifications)
- .hero-illustration, .glass-card, .paper-stack (hero section)
- .cta-group, .cta-input, .cta-btn (code input)
- .tag (pills)
- .fade-in, .gradient-text (animations)
- .navbar, .nav-link (navigation)
- .section-label, .section-title, .section-desc (section headers)

### LOW: Tailwind Config Conflict

index.html line 13: `primary: '#2563EB'` — Tailwind's `bg-primary`, `text-primary` etc. all resolve to blue.
After integration, this must change to `#0D9488` (teal) or be removed if Tailwind utility usage is replaced by design tokens.
